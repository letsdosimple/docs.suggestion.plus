# API Reference

:::note
The Suggestion Plus API is currently in development. This documentation covers the planned endpoints and will be updated as features are released.
:::

## Base URL

```
https://api.suggestion.plus/v1
```

## Authentication

API requests require an API key. Generate one from your dashboard settings.

```
Authorization: Bearer YOUR_API_KEY
```

## Endpoints

### List Inboxes

```http
GET /inboxes
```

Returns all inboxes for the authenticated account.

**Response:**
```json
{
  "inboxes": [
    {
      "id": "inbox_123",
      "name": "Team Feedback",
      "description": "Anonymous team feedback",
      "created_at": "2026-06-01T00:00:00Z",
      "suggestion_count": 42
    }
  ]
}
```

### Get Inbox

```http
GET /inboxes/:id
```

Returns details for a specific inbox.

### Create Inbox

```http
POST /inboxes
```

**Body:**
```json
{
  "name": "New Inbox",
  "description": "Optional description"
}
```

### List Suggestions

```http
GET /inboxes/:id/suggestions
```

Returns suggestions for an inbox, sorted by newest first.

### Submit Suggestion

```http
POST /inboxes/:id/suggestions
```

Submit a suggestion to a public inbox. No authentication required.

**Body:**
```json
{
  "content": "Your anonymous suggestion"
}
```

## Rate Limiting

- 60 requests per minute for authenticated endpoints
- 10 requests per minute for public suggestion submission

## Errors

The API uses standard HTTP status codes. Errors return a JSON object with an `error` field:

```json
{
  "error": "not_found",
  "message": "Inbox not found"
}
```
